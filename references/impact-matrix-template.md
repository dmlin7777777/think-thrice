# Impact Matrix Template

铁律二触发时，按此模板输出影响评估：

```
单向门影响评估
═══════════════════════════════════════

直接改动
  [文件路径] — [改动内容简述]
  [接口名称] — [签名变更说明]

下游影响
  [消费者1] — [影响方式]
  [消费者2] — [影响方式]
  （grep 确认，不编造）

回滚成本：🔴高 / 🟡中 / 🟢低
  理由：[为什么是这个评级]

建议
  □ 现在做 — [理由]
  □ 开新分支 `feature/xxx` — [理由]
  □ 记 inbox，条件成熟再做 — [触发条件]
  □ 不接 — [为什么当前方案已足够]
```

### 回滚成本评级标准

| 级别 | 标准 | 示例 |
|------|------|------|
| 🔴 高 | 改数据库 schema、API 签名、全局状态；回滚需数据迁移 | 删除某张表字段 |
| 🟡 中 | 改核心逻辑但接口不变；回滚需回归测试 | 重构支付计算 |
| 🟢 低 | 改 UI/文案/独立新文件；回滚只需 checkout | 加一个独立组件 |
